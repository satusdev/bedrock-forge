# Fix All Project Issues

**Date:** 2026-03-17 **Status:** IN PROGRESS

## Task

Fix all issues preventing Docker container startup and runtime correctness.

## Root Causes (Priority Order)

### P0 — Container crash-loop

1. `prisma` is in devDependencies — pruned by `pnpm prune --prod` in Dockerfile;
   `migrate deploy` in entrypoint fails
2. `prisma.config.ts` requires `ts-node` at runtime to be loaded by Prisma CLI —
   also pruned
3. No Prisma migrations directory — `migrate deploy` has nothing to run

### P1 — Silent data corruption

4. Worker `EncryptionService` uses `Buffer.from(key.slice(0,32), 'utf8')` — API
   uses `Buffer.from(hexKey, 'hex')` — different binary keys, credentials
   encrypted by API cannot be decrypted by Worker
5. Web auth: API returns `{ accessToken, refreshToken }` (camelCase) — frontend
   reads `res.access_token` (snake_case) — login always fails
6. Web auth: API login response does NOT include a `user` object — frontend
   reads `res.user` — undefined
7. API refresh DTO expects `{ refreshToken }` — frontend sends
   `{ refresh_token }` — validation fails

### P2 — Field name mismatches (web pages vs schema)

8. `BackupsPage`: `file_size` → `size_bytes`, refs `label` (not in schema)
9. `MonitorsPage`: `is_active` → `enabled`, `uptime_percentage` → `uptime_pct`
10. `DomainsPage`: `domain.domain` → `domain.name`
11. `ServersPage`: `ssh_username` → `ssh_user`
12. SSH pool: `this.config.host as unknown as number` → NaN serverId, pool never
    reused

### P3 — Security & code quality

13. Command injection in WHOIS processor
14. Unused imports, dead variables
15. `require('fs')` in processors
16. Backup restore branch missing script push
17. tsconfig deprecation warnings
