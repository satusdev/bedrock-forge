## Issues Fixed

### CRUD Fixes

- ProjectsPage: STATUS_OPTIONS mismatch — `pending/suspended/cancelled` not in
  Prisma enum; `archived` missing
- BackupsPage: No delete capability (backend DELETE endpoint already existed)
- MonitorsPage: No edit dialog (backend PUT already existed; only toggle
  available in UI)
- DomainsPage: No edit/update dialog (backend PUT already existed)
- ServersPage: No pagination/search (backend also extended)
- DashboardPage: `(clientsData as any)?.total` type cast + missing quick actions

### UI Improvements

- LoginPage: Replace raw `<input>/<button>` with shadcn Input/Button/Card;
  password toggle; loading spinner
- Sidebar: Responsive collapse (icon-only on collapse), dark mode toggle, user
  info footer, polished logo
- Header: Sidebar toggle button + breadcrumbs from route + shadcn Button for
  sign out
- AppLayout: Share collapse state via ui.store.ts
- New shadcn components: separator, switch, sheet, skeleton, tooltip

### Global SSH Key

- Backend: `SettingsService.setEncrypted()/getDecrypted()` — AES-256-GCM
  encrypted settings
- Backend: Dedicated `PUT/GET /settings/ssh-key` endpoint (GET returns
  `{has_key: boolean}` — never exposes key value)
- Backend: `CreateServerDto.ssh_private_key` made @IsOptional()
- Backend: `ServersService.testConnection()` falls back to global key when
  server has no per-server key
- Backend: `SettingsModule` imports `EncryptionModule`; `ServersModule` imports
  `SettingsModule`
- Frontend: SettingsPage — dedicated "Global SSH Key" card above generic
  settings
- Frontend: ServersPage — SSH key field is optional; hint text references global
  key setting
